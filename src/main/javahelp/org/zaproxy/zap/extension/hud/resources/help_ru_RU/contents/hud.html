The HUD (визуальный интерфейс)

HUD — это совершенно новый интерфейс, который переносит информацию и функции из ZAP в ваш браузер.

Параметры командной строки

HUD одинаково хорошо работает с ZAP как в режиме рабочего стола, так и в режиме демона.
Поскольку многие люди используют ZAP в режиме демона для автоматизированного тестирования, HUD по умолчанию отключен в этом режиме.
Надстройка HUD добавляет новые параметры командной строки ZAP, которые включают HUD в режиме демона и запускают браузер. предварительно настроен для проксирования через ZAP и игнорирования ошибок сертификата:
  • -hud : запустите браузер с включенным HUD, требуется только в том случае, если вы не хотите указывать браузер или URL-адрес
  • -hudbrowser <browser> : браузер, который будет использовать ZAP, либо Firefox (по умолчанию), либо Chrome.
  • -hudurl <url> : URL-адрес, который будет открыт в браузере, по умолчанию нет
Любая комбинация этих параметров запустит браузер с включенным HUD.

Вы также можете включить HUD в режиме демона без запуска браузера, используя параметр командной строки ZAP: 
-config hud.enabledForDaemon=true

Вовлечены

ZAP — это общественный проект, поэтому мы всегда рады услышать от всех, кто хотел бы внести свой вклад, просто отправьте сообщение в группу ZAP HUD

Предостережения

Это еще рано для этого дополнения :) Некоторые известные проблемы включают в себя:
  • Доступен только ограниченный набор функций ZAP.
  • Мы все еще используем Vue.js в режиме разработки, поэтому не можем применить достаточно сильный CSP.
  • Код JavaScript должен быть правильно отформатирован и проверен.
  • Firefox был протестирован больше, чем Chrome, но оба должны работать.
  • Код для поддержки HUD на нескольких вкладках браузера очень новый, поэтому может содержать ошибки.
  • В частности, не закрывайте первую вкладку в Firefox, иначе HUD перестанет работать.
